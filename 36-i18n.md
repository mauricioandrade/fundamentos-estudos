# 🌍 Internacionalização (i18n e l10n)

> Você fez uma app. Funciona perfeito em pt-BR. Aí o cliente quer vender pra Argentina, EUA e Japão. Trocar textos? Fácil. Lidar com **moedas, datas, números, ordem de leitura, plurais** em 4 idiomas? Aí complica. **i18n** é a arte de fazer apps que se adaptam.

---

## 🎯 i18n vs l10n

Termos confusos. Diferença simples:

| Termo | Vem de | Significa |
|---|---|---|
| **i18n** | "internationalization" (i + 18 letras + n) | **Preparar** o código pra ser traduzível |
| **l10n** | "localization" (l + 10 letras + n) | **Adaptar** pra um locale específico |

> 💡 **Em palavras simples**:
> - **i18n**: você programa pensando que vai ser traduzido (sem hardcode de textos, formatos)
> - **l10n**: você cria os arquivos de tradução pra cada idioma

i18n vem antes de l10n. Sem i18n, l10n é impossível.

## 🗺️ O Que Muda Entre Locales

Não é só o idioma. Pra fazer i18n direito, considere:

### Texto

```
pt-BR: "Olá mundo"
en-US: "Hello world"
es-AR: "Hola mundo"
ja-JP: "こんにちは世界"
ar-SA: "مرحبا بالعالم"  ← direção: direita pra esquerda!
```

### Datas

```
pt-BR: 28/04/2026 (dia/mês/ano)
en-US: 4/28/2026 (mês/dia/ano)
ja-JP: 2026/4/28 (ano/mês/dia)
ISO 8601: 2026-04-28 (universal)
```

### Números

```
pt-BR: 1.234.567,89
en-US: 1,234,567.89
de-DE: 1.234.567,89
fr-FR: 1 234 567,89   (espaço como separador)
```

### Moedas

```
pt-BR: R$ 1.234,56
en-US: $1,234.56
ja-JP: ¥1,234 (sem decimal!)
ar-SA: ر.س 1,234.56
```

### Plurais

Inglês tem 2 formas (1 dog / 2 dogs). Português também (1 cachorro / 2 cachorros).

**Mas:**
- 🇷🇺 **Russo** tem 4 formas plurais
- 🇸🇦 **Árabe** tem 6 formas
- 🇯🇵 **Japonês** geralmente não distingue

```
0 produtos / 1 produto / 2 produtos / 21 produto / 22 produtos (russo é doido)
```

### Direção do texto

- 🇸🇦 **Árabe, Hebraico**: RTL (right-to-left)
- 🌏 **Maioria**: LTR (left-to-right)
- 🇯🇵 **Japonês tradicional**: vertical

### Ordens de nome

```
Ocidente: Nome Sobrenome → "Mauricio Andrade"
Hungria, Japão: Sobrenome Nome → "Andrade Mauricio"
```

### Formatos de endereço

EUA: rua → cidade → estado → ZIP
Brasil: rua → bairro → cidade → estado → CEP
Japão: prefeitura → cidade → distrito → bloco → casa

### Formatos de telefone

```
Brasil: (11) 98765-4321
EUA: (555) 123-4567
Internacional: +55 11 98765-4321
```

## 🏗️ Como Fazer i18n na Prática

### 1. Externalize todos os textos

```javascript
// ❌ Texto hardcoded
<button>Salvar</button>
<p>Bem-vindo, {nome}!</p>

// ✅ Externalizado
<button>{t('actions.save')}</button>
<p>{t('welcome.message', { nome })}</p>
```

### 2. Estrutura de arquivos de tradução

```
locales/
├── pt-BR/
│   ├── common.json
│   ├── home.json
│   └── checkout.json
├── en-US/
│   ├── common.json
│   ├── home.json
│   └── checkout.json
└── es-AR/
    ├── common.json
    ├── home.json
    └── checkout.json
```

Exemplo `pt-BR/common.json`:
```json
{
  "actions": {
    "save": "Salvar",
    "cancel": "Cancelar",
    "delete": "Excluir"
  },
  "welcome": {
    "message": "Bem-vindo, {nome}!"
  }
}
```

### 3. Use ICU MessageFormat pra plurais

ICU é o padrão internacional pra mensagens complexas:

```
{count, plural,
  =0 {Nenhum produto}
  one {# produto}
  other {# produtos}
}
```

A biblioteca escolhe a forma certa baseada na regra do idioma.

## 🛠️ Bibliotecas por Stack

### JavaScript / TypeScript

#### React: `react-i18next` ou `next-intl`

```typescript
import { useTranslation } from 'react-i18next';

function Componente() {
  const { t } = useTranslation();
  return <button>{t('actions.save')}</button>;
}
```

#### Vue: `vue-i18n`
#### Angular: `@angular/localize` (nativo)
#### Vanilla JS: `Intl` API (nativa do browser!)

### Java (Spring)

```java
// messages_pt_BR.properties
welcome.message=Bem-vindo, {0}!

// messages_en.properties
welcome.message=Welcome, {0}!
```

```java
@Autowired
MessageSource messageSource;

String mensagem = messageSource.getMessage(
    "welcome.message",
    new Object[]{nome},
    LocaleContextHolder.getLocale()
);
```

### Python

#### Django: i18n built-in
```python
from django.utils.translation import gettext as _

mensagem = _("Bem-vindo, %(nome)s!") % {'nome': nome}
```

#### FastAPI: `babel` ou `python-fluent`

### Go: `go-i18n` (nicksnyder)
### .NET: built-in com `IStringLocalizer`
### Ruby on Rails: I18n built-in (excelente)

## 🌐 API Intl (JavaScript Nativo)

Browsers modernos têm formatadores embutidos. **Use isso** ao invés de bibliotecas pesadas pra coisas simples.

### Datas

```javascript
const data = new Date();

new Intl.DateTimeFormat('pt-BR').format(data);
// "28/04/2026"

new Intl.DateTimeFormat('en-US').format(data);
// "4/28/2026"

new Intl.DateTimeFormat('pt-BR', {
  dateStyle: 'long',
  timeStyle: 'short'
}).format(data);
// "28 de abril de 2026 às 14:30"
```

### Números

```javascript
new Intl.NumberFormat('pt-BR').format(1234567.89);
// "1.234.567,89"

new Intl.NumberFormat('en-US').format(1234567.89);
// "1,234,567.89"
```

### Moedas

```javascript
new Intl.NumberFormat('pt-BR', {
  style: 'currency',
  currency: 'BRL'
}).format(1234.56);
// "R$ 1.234,56"

new Intl.NumberFormat('ja-JP', {
  style: 'currency',
  currency: 'JPY'
}).format(1234.56);
// "¥1,235" (japonês arredonda automaticamente)
```

### Plurais

```javascript
const pr = new Intl.PluralRules('pt-BR');
pr.select(0);  // "other"
pr.select(1);  // "one"
pr.select(2);  // "other"

const pr_ru = new Intl.PluralRules('ru');
pr_ru.select(1);  // "one"
pr_ru.select(2);  // "few"
pr_ru.select(5);  // "many"
pr_ru.select(11); // "many"
pr_ru.select(21); // "one"  (russo é doido!)
```

### Datas relativas

```javascript
const rtf = new Intl.RelativeTimeFormat('pt-BR');
rtf.format(-1, 'day');  // "há 1 dia"
rtf.format(2, 'hour');   // "em 2 horas"
```

### Comparação de strings (collation)

```javascript
const collator = new Intl.Collator('sv'); // sueco
['a', 'z', 'å'].sort(collator.compare);
// ['a', 'z', 'å'] (correto em sueco — 'å' vem depois de 'z')
```

## 🎯 Padrões e Boas Práticas

### 1. Use locale tags BCP 47

```
pt-BR    ← português do Brasil
pt-PT    ← português de Portugal
en-US    ← inglês americano
en-GB    ← inglês britânico
zh-Hant  ← chinês tradicional
zh-Hans  ← chinês simplificado
```

### 2. Detecção de idioma

```javascript
// Browser
const idioma = navigator.language;  // ex: "pt-BR"
const idiomas = navigator.languages; // ex: ["pt-BR", "pt", "en"]

// HTTP Header (servidor)
Accept-Language: pt-BR,pt;q=0.9,en;q=0.8
```

> 💡 **Sempre** dê opção do usuário trocar manualmente. Detecção automática erra.

### 3. Persistência da escolha

```javascript
// Cookie ou localStorage
localStorage.setItem('user_locale', 'en-US');
```

### 4. URLs com locale

```
https://meusite.com/pt-BR/produtos
https://meusite.com/en-US/products
```

✅ Bom pra SEO
✅ Compartilhável (link já vem no idioma)

### 5. Fallback chain

```
Pediu: zh-CN
Tentamos: zh-CN → zh → en (default)
```

### 6. RTL precisa de cuidado especial

```css
/* CSS direcional */
margin-left: 10px;  /* hardcoded */

/* CSS lógico (RTL-friendly) */
margin-inline-start: 10px;  /* "início" — esquerda em LTR, direita em RTL */
```

```html
<html dir="rtl" lang="ar">
```

## 🚨 Anti-Patterns

### 1. Concatenar strings traduzidas

```javascript
// ❌ Quebra em outros idiomas
const msg = t('youHave') + ' ' + count + ' ' + t('messages');

// ✅ Use placeholders
const msg = t('youHaveMessages', { count });
```

```json
{
  "youHaveMessages": "Você tem {count, plural, one {# mensagem} other {# mensagens}}"
}
```

### 2. Hardcode de formato de data/moeda

```javascript
// ❌
const formatado = `R$ ${valor.toFixed(2).replace('.', ',')}`;

// ✅
const formatado = new Intl.NumberFormat('pt-BR', {
  style: 'currency', currency: 'BRL'
}).format(valor);
```

### 3. Imagens com texto

```html
<!-- ❌ Botão é uma imagem PNG com "Comprar" embutido -->
<img src="botao-comprar.png" />

<!-- ✅ -->
<button>{t('buy')}</button>
```

### 4. Layout fixo que quebra com texto longo

Alemão é ~30% mais longo que inglês. Tailândes ocupa mais altura. **Design flexível**.

```
Inglês: "Save" (4 chars)
Alemão: "Speichern" (9 chars)
```

### 5. Imagens culturalmente inadequadas

- 🇨🇳 Vermelho = sorte (China) / perigo (ocidente)
- 🇸🇦 Não use mulheres em roupas curtas
- 🇮🇳 Não use vacas em contextos de carne
- 🦊 Em alguns países, gestos com a mão têm significados muito diferentes

### 6. Esquecer de SEO multi-idioma

```html
<!-- Diga ao Google que existem versões em outros idiomas -->
<link rel="alternate" hreflang="pt-BR" href="/pt-BR/produtos" />
<link rel="alternate" hreflang="en-US" href="/en-US/products" />
<link rel="alternate" hreflang="x-default" href="/produtos" />
```

## 🔄 Workflow de Tradução

Pra projetos sérios, traduções não são feitas pelo dev.

```
1. Dev externaliza textos (chaves + texto base em pt-BR)
2. Exporta JSON/CSV pra plataforma de tradução
3. Tradutores profissionais traduzem
4. Reimporta no projeto
5. Testa em cada locale
```

### Plataformas de tradução

- **Lokalise** (popular)
- **Crowdin**
- **Phrase**
- **Weblate** (open source)
- **Transifex**

### Tradução automática

ChatGPT, Claude, DeepL fazem traduções razoáveis pra prototipagem. **Mas pra produção, sempre revisão humana** — IA erra contexto e idiomas com nuances.

## 🧪 Testando i18n

### Pseudo-localização

Antes de ter traduções reais, pseudo-traduza:

```
"Hello World" → "[Ħëļļö Ŵöŕļđ ñ§!]"
```

Isso ajuda detectar:
- Strings hardcoded (não foram pseudo-traduzidas)
- Layout que quebra com texto mais longo
- Caracteres especiais não suportados (emojis, acentos exóticos)

### Snapshot por locale

Tira screenshots em cada idioma e compara visual.

### Long text test

Use idioma com strings longas (alemão, finlandês) ou idiomas RTL (árabe, hebraico) pra estressar layout.

## ✅ Resumo da página

- **i18n** = preparar pra tradução; **l10n** = adaptar pra um locale específico
- Não é só texto: **datas, números, moedas, plurais, direção, ordens** mudam
- **Externalize** todos os textos em arquivos de tradução
- Use **ICU MessageFormat** pra plurais e mensagens complexas
- **Intl API** (JS nativo) cobre 80% das necessidades de formatação
- Tags **BCP 47** (`pt-BR`, `en-US`) padronizam locales
- **CSS lógico** (`margin-inline-start`) facilita RTL
- Plataformas (**Lokalise, Crowdin**) gerenciam tradução em times
- **Pseudo-localização** detecta problemas antes da tradução real
- **NUNCA** concatene strings — use placeholders sempre
